# 코딩테스트 문제 풀이 기록

## 1. 문제 유형
<!-- 예: DP, BFS/DFS, 그리디, 정렬 등 -->
- 구현, 정렬, 필터링

## 2. 배운 점
<!-- 문제를 풀면서 새롭게 알게 된 내용, 활용한 알고리즘/자료구조 -->
- 문자열 기준을 정수 인덱스로 매핑해 유연한 조건 처리가 가능함
- Stream을 사용하면 필터링과 정렬을 더 간결하게 표현할 수 있음
- Comparator.comparingInt를 사용하면 기준 정렬을 명확하게 지정할 수 있음

## 3. 개선할 점
<!-- 문제를 풀면서 아쉬웠던 점, 다음에 더 잘할 수 있는 부분 -->
- 원본 배열을 직접 리스트에 담는 것은 사이드 이펙트를 낳을 수 있음 -> 방어적 복사가 필요할 때가 있음
- 매핑 키를 문자열로 유지할 경우 오타에 취약함 -> enum 활용 고려
- Stream API의 성능 특성을 더 이해하면 큰 데이터 처리에서 올바른 선택을 할 수 있음

## 4. 추가 연습이 필요한 부분
<!-- 유사 문제나 더 연습이 필요한 알고리즘/자료구조 또는 부족한 부분 -->
- Stream의 filter / sorted / map / toArray 패턴
- Comparator 활용
- 배열을 다룰 때 shallow copy / deep copy 차이
- 필드 기반 정렬 문제 (여러 기준 정렬)

## 5. 문제에서 가장 어려웠던 점
<!-- 설계, 구현, 디버깅 등 어려웠던 부분 상세히 기록 -->
- 문자열로 주어진 ext, sort_by를 인덱스로 변환하는 로직
- 배열 기반 자료 구조를 정렬하면서 side-effect 없이 처리하는 방법
- 필터링 -> 정렬 -> 변환 과정을 안정적인 코드로 조직하는 점

# 6. 다른 접근 방식

- Stream API 사용
- enum 기반 인덱스 매핑
- 배열 복사 후 정렬하는 방식
