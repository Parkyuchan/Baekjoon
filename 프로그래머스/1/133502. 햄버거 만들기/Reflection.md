# 코딩테스트 문제 풀이 기록

## 1. 문제 유형
<!-- 예: DP, BFS/DFS, 그리디, 정렬 등 -->
- 시뮬레이션
- 패턴 매칭
- 스택 활

## 2. 배운 점
<!-- 문제를 풀면서 새롭게 알게 된 내용, 활용한 알고리즘/자료구조 -->
- List.remove() 연속 호출은 시간 복잡도가 커서 대량 데이터에서는 적합하지 않음을 인지
- 패턴 탐지 문제에서는 스택을 사용하는 것이 더 효율적
- 인덱스 조작 기반 구현은 제거 연산이 동반될 때 매우 불안정

## 3. 개선할 점
<!-- 문제를 풀면서 아쉬웠던 점, 다음에 더 잘할 수 있는 부분 -->
- 대규모 입력에서는 반드시 시간 복잡도를 고려
- remove 기반 풀이 지양
- 인덱스 기반 처리보다 스택 기반 선호
- 조건문 패턴 탐지는 데이터가 많을 때 O(N)으로 줄이는 방법 우선 고려

## 4. 추가 연습이 필요한 부분
<!-- 유사 문제나 더 연습이 필요한 알고리즘/자료구조 또는 부족한 부분 -->
- 스택을 활용한 패턴 제거 문제
- O(N^2) -> O(N)으로 최적화하는 연습
- 인덱스 기반 탐색에서 shift가 발생할 때 구조적 문제 분석

## 5. 문제에서 가장 어려웠던 점
<!-- 설계, 구현, 디버깅 등 어려웠던 부분 상세히 기록 -->
- 패턴 삭제 후 인덱스가 변하면서 발생하는 버그
- remove 연속 호출 시 성능 문제
- 이전 지점을 기억하는 로직이 복잡해지는 것

# 6. 다른 접근 방식
<!-- 다른 풀이 방법이 있었다면 기록 -->
- 스택 사용 (채택 완료!)
- 배열 기반 스택 사용(더 빠름)
- KMP처럼 패턴 매칭 알고리즘도 이론적으로 가능하나 오히려 과함
- 투 포인터 기반 시뮬레이션도 가능하지만 구현 난이도 증가
