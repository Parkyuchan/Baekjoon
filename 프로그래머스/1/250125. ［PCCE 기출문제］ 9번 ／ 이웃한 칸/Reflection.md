# 코딩테스트 문제 풀이 기록

## 1. 문제 유형
<!-- 예: DP, BFS/DFS, 그리디, 정렬 등 -->
- 단순 구현, 시뮬레이션

## 2. 배운 점
<!-- 문제를 풀면서 새롭게 알게 된 내용, 활용한 알고리즘/자료구조 -->
- 기준 위치의 값을 변수에 저장해두면 매 반복마다 board[h][w]를 조회하지 않아도 되어 미세하게 효율이 좋아짐.
- 방향 이동 배열(dh, dw)을 사용하면 상·하·좌·우 탐색을 일정한 패턴으로 처리할 수 있음.

## 3. 개선할 점
<!-- 문제를 풀면서 아쉬웠던 점, 다음에 더 잘할 수 있는 부분 -->
- color 기준 4칸만 반복하는 것이기 때문에 반복문을 쓰지 않고 직접 비교 가능
- board의 크기를 반복적으로 조회(board.length 등)하는 부분은 로컬 변수에 담아두면 코드 가독성과 접근 비용 소폭 향상

## 4. 추가 연습이 필요한 부분
<!-- 유사 문제나 더 연습이 필요한 알고리즘/자료구조 또는 부족한 부분 -->
- 2D 좌표 문제에서 x, y, row, column을 혼동하지 않는 연습
- 방향 배열 활용 패턴(dh, dw)을 다양한 문제에 적용하는 연습

## 5. 문제에서 가장 어려웠던 점
<!-- 설계, 구현, 디버깅 등 어려웠던 부분 상세히 기록 -->
- dh, dw 배열에서 h -> x좌표, w -> y좌표로 적용할 때 축을 혼동하여 계산 실수할 수 있었음

## 6. 다른 접근 방식
<!-- 다른 풀이 방법이 있었다면 기록 -->
- 반복문 제거하고 상/하/좌/우 직접 비교 (코드는 길어지지만 분기 구조 명확)
- board가 숫자나 문자라면 String 비교 대신 더 빠른 원시 타입 비교 방식으로 구현 가능
