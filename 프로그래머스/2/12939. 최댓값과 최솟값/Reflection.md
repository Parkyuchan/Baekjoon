# 코딩테스트 문제 풀이 기록

## 1. 문제 유형
<!-- 예: DP, BFS/DFS, 그리디, 정렬 등 -->
- 단순 구현

## 2. 배운 점
<!-- 문제를 풀면서 새롭게 알게 된 내용, 활용한 알고리즘/자료구조 -->
- Arrays.sort의 시간 복잡도: O(n log n) (primitive 배열)
- Arrays.sort의 시간 복잡도: O(n)
- 컬렉션을 사용하여 Collection.max 또는 Collection.min을 활용해도 되지만, int -> Integer 박싱으로 오버헤드가 발생
- 최솟값과 최댓값만 필요할 경우, 정렬 없이 한 번의 순회로도 충분히 구할 수 있음 (시간 복잡도 O(n))

## 3. 개선할 점
<!-- 문제를 풀면서 아쉬웠던 점, 다음에 더 잘할 수 있는 부분 -->
- 배열과 컬렉션 중 선택할 때 장단점을 고려해야 함
- 불필요한 정렬 대신, 최솟값과 최댓값을 바로 계산하도록 구현하는 것이 효율적
- 메서드를 분리하여 코드 가독성과 재사용성을 높이는 습관 필요

## 4. 추가 연습이 필요한 부분
<!-- 유사 문제나 더 연습이 필요한 알고리즘/자료구조 또는 부족한 부분 -->
- 최솟값/최댓값, 평균/합계 등 단순 구현 문제에서 효율적으로 처리하는 패턴 연습
- primitive와 객체 타입 사용 시 성능 차이 이

## 5. 문제에서 가장 어려웠던 점
<!-- 설계, 구현, 디버깅 등 어려웠던 부분 상세히 기록 -->
- 처음에는 배열 정렬로 최솟값과 최댓값을 구했지만, 불필요한 연산임을 깨닫는 과정

## 6. 다른 접근 방식 (채택 완료)
<!-- 다른 풀이 방법이 있었다면 기록 -->
- 정렬이 아닌, 한 번의 순회로 최솟값과 최댓값을 계산
	- 예상 시간 복잡도: O(n)
	- 예상 공간 복잡도: O(1)
