# 코딩테스트 문제 풀이 기록

## 1. 문제 유형
<!-- 예: DP, BFS/DFS, 그리디, 정렬 등 -->
- 스택 / 단순 구현

## 2. 배운 점
<!-- 문제를 풀면서 새롭게 알게 된 내용, 활용한 알고리즘/자료구조 -->
- 처음에는 익숙한 방식인 Stack을 떠올렸지만, 괄호의 개수만으로도 해결 가능하다는 점을 확인함
- 추가 자료구조 없이 int 변수만으로도 구현할 수 있어 더 간단하고 효율적인 풀이가 가능함
- Stack을 쓰면 O(n)의 공간 복잡도, 카운터는 O(n)의 공간 복잡도 발생
	- 공간 복잡도 월등히 상승
- Stack 방식 비용
	1. push() 시 객체 (boxing) 또는 내부 접근 필요
	2. 참조 타입 저장 -> 메모리 접근 비용 증가
	3. 스택 내부 배열 크기 증가 체크
	4. 메서드 호출 비용 증가
	5. GC 대상 객체 증가 가능성

## 3. 개선할 점
<!-- 문제를 풀면서 아쉬웠던 점, 다음에 더 잘할 수 있는 부분 -->
- 풀이 시작 시 익숙한 방식(Stack 등)에만 의존하지 않고, 더 단순하고 효율적인 방법이 있는지 먼저 탐색할 필요가 있음

## 4. 추가 연습이 필요한 부분
<!-- 유사 문제나 더 연습이 필요한 알고리즘/자료구조 또는 부족한 부분 -->
- 완전 탐색 및 단순 구현 문제에서 불필요한 자료구조 사용을 줄이고, 최소한의 연산으로 해결하는 패턴 연

## 5. 문제에서 가장 어려웠던 점
<!-- 설계, 구현, 디버깅 등 어려웠던 부분 상세히 기록 -->
- 이미 알고 있는 방식에 매몰되어 더 효율적인 풀이 방향을 떠올리는 것이 어려웠음

## 6. 다른 접근 방식
<!-- 다른 풀이 방법이 있었다면 기록 -->
